# Contributing to CONTRIBUTING.md
Todas las contribuciones, informes de errores, correcciones de errores, mejoras en la documentación, mejoras y ideas son bienvenidas.
Todas las contribuciones:
Esta aplicacion tiene como objetivo calcular el interès simple utilizando la formula:
interés simple = p*t*r

El objetivo de este viaje es alojar el codigo de muestra para calcular el interés simple en GitHub en un nuevo repositorio.
No solo alojar el script, sino que también seguir las mejores prácticas y crear documentos de apoyo para el proyecto de código abierto.

Informe de errores:
Si encuentra algun error, por favor seguir las siguientes instrucciones:
1.Verificar que realmente el error ocurra.
2. Revisar si ya existe in Issue abierto sobre el mismo error.
3.Asegurese que esta usando la version reciente del proyecto.

Correccion de errores:
Para corregir el error debo considerar:
.Que el calculo del interes es incorrecto.
.El monto total no coincide con la formula.
.No se validan correctamente los datos.
.Existen errores en la consola.

Procesos para corregir el error:
1.No se deben hacer correcciones directamente en main.Debo crear una nueva rama.
2.Analizar el problema
Antes de modificar el codigo debo:
.Identificar la causa del error.
.Revisar la formula matematica.
.Verificar validaciones existentes.
3.Se deben probar diferentes escenarios.
4.Realizar un commit claro.
5.Crear un pull request que debe especificar con claridad:
.Descripcion del error.
.Causa del problema.
.Como se soluciono.
.Evidencia de las pruebas.
.Confirmacion de que no afecta otras funciones.

Mejoras de la documentacion:
Buenas practicas de documentacion:
-usar lenguaje claro y sencillo.
-mejorar el README.
-agregar ejemplos.
-corregir errores ortograficos.
-explicar la formula.

 Mejoras:
Mejoras en el diseño:
-mejorar el interfaz.
-optimizar experiencias del usuario.

Estandares de codigo:
.usar nombres descriptivos("Ingresa el capital","Ingresa la tasa de interés por año","Ingresa el período de tiempo en años")
.separar HTML,CSS y JavaScrip.
.documentar funcionalidades importantes.
.mantener estructura organizativa.

Restricciones:
No esta permitido:
.Realizar push directo a main.
.Subir archivos innecesarios.
.Eliminar trabajo de otros desarrolladores.
.Enviar codigo sin probar.

Ideas:
La precision es fundamental en calculos financieros.Trabajaremos con responsabilidad,claridad y buenas practicas.
Este documento garantiza que el proyecto mantenga estandares academicos y profesionales adecuandos.
Toda contribucion debe respetar estos lineamientos para asegurar calidad,precision matematica y organizacion del repositorio.
